# `policy_facets/`

Everything in this folder is **derived** from `<domain>/input/policy_docs/`.
Nothing here is an original policy document. The canonical analyst-authored
truth lives in `<domain>/input/policy_docs/`.

## What's here

- `input-index.yaml` — metadata only: per-source git SHA + md_quality score.
  Generated by `/index-inputs`.
- `compressed/<rel>.md` — caveman-compressed (token-reduced) mirrors of
  `input/policy_docs/<rel>.md`. Generated by `/compress-inputs` and consumed
  by downstream skills (`/extract-ruleset`, `/update-ruleset`) as the canonical
  *content* source.
- `computations/<rel>.md.yaml` — per-source YAML lists of
  `{heading, summary, tags, computations}` section blocks. Generated by
  `/extract-computations` and consumed by `/suggest-target-ruleset`,
  `/create-skeleton`, `/create-ruleset-groups`, `/create-ruleset-modules`,
  `/extract-sample-rules`, `/refine-guidance`.
- `.compress-manifest.yaml`, `.computations-manifest.yaml` — sync state
  (source SHA → derived file). Do not edit; do not delete between runs unless
  you intend a full rebuild.
- `.compress-plan.tmp`, `.extract-plan.tmp` — transient work plans. Present
  only mid-run; removed by the matching `--finalize`.

## Rules

- **Do NOT edit any file in this folder manually.** All files are derived.
  Edit the source under `<domain>/input/policy_docs/` instead and re-run
  `/index-inputs <domain>` (which orchestrates the downstream regeneration).
- **Do NOT cite `compressed/<rel>.md` as policy text.** The compressed form is
  a lossy paraphrase optimized for AI input — it is not a quotable source.
  Quote `input/policy_docs/<rel>.md` instead.
- **Path-key convention:** `input-index.yaml` keys are
  `input/policy_docs/<rel>.md` (source paths). Skills that need *content* read
  the parallel path under `compressed/<rel>.md`. Skills that need *section
  data* read the parallel path under `computations/<rel>.md.yaml`. See the
  "Index path keys vs content reads" section in `xl-plugin/CLAUDE.md`.

## This file is auto-managed

`/index-inputs` overwrites this `CLAUDE.md` on every run from the template at
`xl-plugin/core/policy_facets_claude.md`. Edit the template, not this copy.
